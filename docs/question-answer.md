# 项目场景问题说明

## Q: 为什么只使用 JSON 存储？

A: 这个项目主要保存账号、鉴权密钥和少量运行配置，不会产生需要数据库承载的大规模结构化数据。保留 SQLite、PostgreSQL、MySQL 或
Git 存储只会增加依赖、配置和维护成本，所以统一使用本地 JSON 文件。

## Q: 每个账号的额度大概是多少？

A: Free 账号额度通常在 0-50 不等，Plus/Team 账号大约 120 左右，Pro 账号无限。

## Q: 每个账号能存活多久？

A: 不确定。逆向调用可能会导致 token 很快失效，仅供测试使用。

## Q: 不同 token 的区别是什么？

A: Web 的 at_token 可以用来网页对话；Codex 里的 at_token 可以用来调用 Codex 接口，也可以用来网页对话。Codex 的 at_token
导入本系统后，可以同时使用网页逆向额度和 Codex 接口生图额度。本系统不建议导入 Codex 的 rt_token，刷新 rt 后会导致其他地方的
rt 失效。

## Q: 为什么次版本移除了很多功能？

A: 没精力维护过多的功能，解决指纹、稳定性、账号存活和接口兼容等核心问题比较好。此版本会优先保留和
2api 直接相关的能力，移除维护成本高但和主线关系不大的功能。
